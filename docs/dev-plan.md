## 第 11.5 阶段：日历页重构（时光回顾）

**前置条件**：第 11 阶段（首页重构）完成后。

**目标**：将日历页从"查看某一天"升级为"寻找并回忆某一天"，打造具备产品辨识度的日历页。

### 设计原则

> 首页讲"今天"，日历讲"那一天"，洞察讲"那些天"。

- 日历页不再只是一个月历工具，而是一页 **「时光回顾」**
- 核心定位：**寻找某一天 → 回忆某一天**（情感型，非功能型）
- 三大页面职责分工：🏠 首页（现在时）· 📅 日历（过去时）· 📊 洞察（完成时）

### 最终布局方案

#### 桌面端 — 左右双栏

```
┌────┬────────────────────────────────────────┬──────────────────────┐
│    │  💭 回忆流（70%）                       │  📅 时间导航（30%）   │
│    │                                        │                      │
│    │  🌱 2026年6月27日                       │  年度热力图           │
│    │  😊 开心                                │  ███ ███ ███ ███    │
│    │                                        │  活跃178天 连续126天  │
│ sb │  今天完成了工作计划，坚持开发             │                      │
│    │  TimeFlow，晚上学习Flutter。            │  ├──────────────┤    │
│    │                                        │  │  2026 · 6    │    │
│    │  ───────── 去年今天 ─────────           │  │  一 二 三 四  │    │
│    │  📖 第一次开始学习 Flutter               │  │  五 六 日     │    │
│    │                                        │  │  22 23 24 25  │    │
│    │  🏷8标签  📝1450字  📍6事件              │  │  26 [27] 28   │    │
│    │                                        │  │  29 30        │    │
│    │  查看完整日记 →                         │  └──────────────┘    │
│    └────────────────────────────────────────┴──────────────────────┘
```

**设计决策（重要）：**
- Sidebar 已在左侧占 `w-56`，为给月历留够空间 → **调换左右比例**
- 回忆流放**左侧 70%**（主内容优先）
- 热力图 + 月历放**右侧 30%**（导航辅助）

| 区域 | 宽度 | 内容 | 行为 |
|------|------|------|------|
| 右侧（30%） | 固定 | 年度热力图 + 月历（月份切换） | 不滚动、不闪烁，作为"遥控器" |
| 左侧（70%） | 自适应 | 当天回忆流（日期标题+摘要+去年今天+统计+入口） | 选日期后动态刷新 |

#### 手机端 — 纵向流式

```
热力图（横向滚动）
  ↓
月历（单月切换，点击日期）
  ↓
当天的回忆流（Memory Timeline 卡片）
```

- 不重新加载页面，上滑浏览
- 日期切换通过月历点击触发

### 需要新建的模块

| 模块 | 说明 | 数据源 |
|------|------|--------|
| **去年今天** | 去年同一天是否有日记，展示摘要 | Supabase 额外查 `DATEADD(year, -1, ...)` |
| **当日摘要** | 当天日记的浓缩文字（首段） | `diary.content`（需解密） |
| **统计摘要** | 标签数 / 字数 / 事件数 / 图片数 | 从 diary 和 tags 计算 |
| **心情** | 😊 开心等表情（可选） | 需数据库新增字段或标签约定 `#心情` |
| **回忆流** | 右侧卡片展示区 | 组合以上模块 |

### 数据结构缺口

| 字段 | 当前 | 方案 |
|------|------|------|
| 心情/表情 | ❌ 无 | 暂无想法，开发时再定 ⏳ |
| 去年今天 | ✅ 已有 | 直接查去年同一天的日记，没有就不显示 ✅ |

### 步骤

- [ ] 重构布局：左侧回忆流 70% + 右侧热力图+月历 30%（桌面 Grid）
- [ ] 月历组件拆为独立模块，支持年份/月份选择联动
- [ ] 热力图复用已有逻辑，移到右侧
- [ ] 实现"去年今天"数据查询（年份偏移 + 日期匹配）
- [ ] 实现日记内容解密预览（复用 `use-diary.ts` 的 `decryptDiary`）
- [ ] 构建回忆流卡片：日期头 + 心情 + 摘要 + 去年今天 + 统计摘要
- [ ] 添加"查看完整日记"按钮跳转
- [ ] 手机端纵向流适配（热力图 → 月历 → 回忆流）

## 第 13 阶段：桌面封装（Tauri 2）

**前置条件**：第 11、12 阶段完成后，进入封装阶段。

**目标**：将 Web 应用打包为 Windows .exe / .msi 安装包。

### 步骤

- [ ] 安装 Rust 工具链（rustup.rs）
- [ ] 安装 Tauri CLI：`npm install -D @tauri-apps/cli`
- [ ] 初始化：`npx tauri init`
  - 配置 `devUrl` → `http://localhost:3000`
  - 配置 `distDir` → `../out`
  - 配置 `beforeBuildCommand` → `npm run build`
- [ ] 处理 basePath：Tauri 环境去掉 `/-` 前缀（非 GitHub Pages 不需要）
  - 方案：通过环境变量或 `process.env.TAURI` 判断
  - 若 `TAURI` 环境变量存在，`basePath` 设为 `""`
- [ ] 跳过 PWA 注册：检测非浏览器环境时跳过 SW 注册
  - 方案：在 `layout.tsx` SW 注册脚本中包一层检测
- [ ] 配置窗口标题、尺寸、图标（复用已有 512x512 PNG）
- [ ] 配置 Tauri 权限（Shell、Notification、GlobalShortcut 等）
- [ ] 添加系统托盘 + 开机自启配置
- [ ] 添加全局快捷键（Ctrl+Alt+D 快速写日记）
- [ ] 配置自动更新（`tauri-plugin-updater`）
- [ ] dev 模式测试：`npx tauri dev`
- [ ] 构建生产包：`npx tauri build` → 输出 `.msi` / `.exe`

### 输出物
- `src-tauri/` 目录（Rust 项目）
- `out/时光轴_0.1.0_x64.msi`（安装包，~10MB）

## 第 14 阶段：安卓封装（Capacitor）

**前置条件**：第 13 阶段完成后，进入此阶段。

**目标**：将 Web 应用打包为 Android .apk 安装包。

### 步骤

- [ ] 安装 Android Studio（需 Java 17+ JDK）
- [ ] 配置 Android SDK 环境变量（ANDROID_HOME）
- [ ] 安装 Capacitor 核心包：`npm install @capacitor/core @capacitor/cli`
- [ ] 添加 Android 平台：`npm install @capacitor/android`
- [ ] 初始化：`npx cap init 时光轴 com.timeaxis.app`
- [ ] 添加 Android 工程：`npx cap add android`
- [ ] 处理 basePath：Capacitor 加载本地文件，同样需要去掉 `/-` 前缀
- [ ] 适配底部安全区：用 `env(safe-area-inset-bottom)` 补充内边距
- [ ] 添加键盘适配插件：`npm install @capacitor/keyboard`
- [ ] 构建 Web 资源：`npm run build` → `npx cap sync`
- [ ] 真机调试：`npx cap run android`
- [ ] 配置应用图标 + 启动屏（Splash Screen）
- [ ] 生成发布版 .apk
- [ ] （可选）上架应用商店

### 输出物
- `android/` 目录（Android Studio 工程）
- `app-release.apk`（安装包，~5-8MB）
